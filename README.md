# h5-pg-pages

本仓库用于归档、存放并发布多个独立的 HTML 页面。这些页面功能简单，不依赖于特定后端服务，适合直接通过静态托管方式访问。

## 项目简介

`h5-pg-pages` 是一个轻量级的静态页面仓库，主要收集和整理一些小型、独立的 HTML 文件。每个页面可以包含独立的样式和脚本，彼此之间没有关联。仓库本身不针对任何具体网站或应用，仅作为页面文件的存储与展示场所。

## 目录结构

```
h5-pg-pages/
├── README.md          # 本说明文件
├── index.html         # 可选：仓库总入口页
├── page1/             # 示例页面目录
│   └── index.html
├── page2/
│   └── index.html
└── ...
```

- 每个子文件夹对应一个独立的页面，文件夹名即页面标识。
- 每个页面文件夹内至少包含一个 `index.html` 文件，作为该页面的入口。
- 页面内资源（CSS、JS、图片等）建议存放在同一文件夹中。

## 页面归档说明

- 所有页面均为独立 HTML 文件，可直接在浏览器中打开使用。
- 页面不依赖外部数据库或服务器端逻辑，所有功能均在前端实现。
- 归档的页面通常为小型工具、演示、示例或临时页面，不保证长期维护或更新。
- 页面内容可能包含实验性或非生产环境代码，使用时请自行评估。

## 维护说明

- 本仓库不定期更新，新增页面或修正已有页面。
- 若发现页面无法正常访问或功能异常，可提交 Issue 说明。
- 欢迎提交 Pull Request 添加新的独立 HTML 页面，但请确保页面不包含恶意代码、不侵犯他人权益。
- 仓库维护者保留对页面内容进行修改或移除的权利。
